What

A playground where you create your own actions triggered by EEG signal patterns.

Why

Having fun making creative real-world actions from brain signals.

Who

Anyone interested in exploring the potential of brainwaves and real-world actions.

Where

Playground

1 - Interact with a 3D scene

Interacts with the 3D scene embedded to the right which could have been a online multiplayer game.

Notice you can use mouse wheel and mouse down to zoom and rotate the scene.

  • Flashes the background color
  • Wiggles (shakes) an element on the page
  • Shows a popup (toast) in the lower right corner
  • Plays an audio clip
  • Talks to the user via the speech API
  • Moves the pin up in the 3D scene and changes its material color to a new random color

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

2 - Interact with videos

Inserts a randomized YouTube video iframe into a given content element.

  • Flashes the background color
  • Wiggles (shakes) an element on the page
  • Shows a popup (toast) in the lower right corner
  • Plays an audio clip
  • Talks to the user via the speech API
  • Changes to a random YouTube video

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

3 - Send via URL

Posts data via an URL to an external web app or REST API.

  • Posts data to the web app https://httpbin.org/post.
  • Gets a response with some metadata and echo of the message sent.

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

4 - Send via WebSocket

Sends a message to a WebSocket host.

  • Sends a message to a host (ex. ws://localhost:8889).
  • Can wait for a specified number of replies, a timeout or just fire and forget.

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

5 - Send via UDP

Sends a message via UDP, which is also used by multiplayer games and streaming services.

  • Sends an UDP message to a host (ex. localhost) at a specific port (ex. 8888).
  • Can wait for a specified number of replies, a timeout or just fire and forget.

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

6 - Send via MQTT

Sends a message to a message queue broker via the Message Queuing Telemetry Transport protocol (MQTT) used by most IoT devices.

  • Sends a message to wss://broker.emqx.io:8084/mqtt on the topic browser/demo.
  • Can wait for a specified number of replies, a timeout or just fire and forget.

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.

7 - Execute shell script

Sends a shell command to a local or remote host via Shell Server.

Browsers cannot use local OS shell scripts, so this request is dispatched to a Shell Server.

  • Sends a shell command c:\code\echo-stuff.cmd to a host on a give port (ex. http://localhost:3000).
  • Returns the output from the shell command.

Click Run to test this or Edit to modify this pattern.
Drag triangle to resize.
