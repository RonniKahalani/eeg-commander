# How to become a contributor
The EEG Commander app is an open-source project and here is an overview of how to become part of the development team. Please start by reading the 
[Code of Conduct](/docs/CODE_OF_CONDUCT.md).

## Core Philosophy
We value **consistent quality contributions**, **good collaboration**, and **long-term commitment** over speed or flashy one-off PRs.

## Qualification Process (Step-by-Step)
- **Initial Contribution** - Submit at least 1–2 meaningful PRs (bug fix, feature, documentation, template, etc.) [1–4 weeks]
- **Consistent Activity** - Multiple high-quality contributions + good communication [4–8 weeks]
- **Community Involvement** - Active in Discord (helping others, discussions, feedback) [Ongoing]
- **Core Team Evaluation** - Reviewed by current maintainers
- **Official Invitation** - Private outreach + offer to join

## Minimum Qualifications to Become a Core Contributor
- **Code / Technical Contributions**: At least 3–5 meaningful merged PRs
- **Quality Standards**: Clean code, good documentation, follows project style
- **Communication**: Responsive, constructive, respectful
- **Reliability**: Completes what they start or communicates blockers
- **Alignment**: Understands and supports the project vision (mental commands + creative playground)

## Nice-to-Haves (big advantage):
- Created popular templates
- Helped other community members
- Improved documentation or infrastructure
- Fixed critical bugs

## Expectations Once Accepted
**Time Commitment** (flexible)
- Contribute regularly (not necessarily every week)
- Review PRs when requested
- Participate in major discussions

**Behavioral Expectations**
- Represent the project professionally
- Follow the Code of Conduct
- Help mentor newer contributors
- Be transparent about availability

**Privileges**:
- @core-contributor Discord role
- GitHub write access (or triage first)
- Influence on roadmap and priorities
- Featured in changelog / credits
